# Contributing to local_enddateaccess

Thank you for your interest in contributing!

## Reporting Bugs
Use the [Bug Report](.github/ISSUE_TEMPLATE/bug_report.yml) template. Include your Moodle and PHP versions.

## Proposing Features
Use the [Feature Request](.github/ISSUE_TEMPLATE/feature_request.yml) template.

## Pull Requests
1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Ensure your code follows the [Moodle Coding Style](https://moodledev.io/general/development/policies/codingstyle).
4. Commit your changes: `git commit -m "feat: your description"`
5. Push to the branch: `git push origin feature/your-feature-name`
6. Open a Pull Request.
